UX หรือ User experience คือคนละอย่างกับ UI หรือ User interface โดย UX คือสิ่งที่เป็นประสบการณ์ของผู้ใช้ อย่างง่ายๆ คือเมื่อผู้ใช้เปิดหน้าเว็ปเราขึ้นมากแล้วรู้ได้ทันทีว่า สิ่งนี้กดได้หรือไม่ หรือว่าเราต้องการให้ผู้ใช้ทำอะไร ส่วน UI คือสิ่งที่แสดงให้ผู้ใช้เห็นว่าหน้าเว็ปของเราน่าสนใจหรือไม่ โดยจะเกี่ยวกับการจัดวาง Template การเลือกสี หรือง่ายๆ ผมก็ได้เห็น UI designที่ทำให้ผมว้าวมากตอนเข้าเว็ปสมัครนี้ครับ




1. คุยกับลูกค้า เรื่องวัตถุประสงค์ของลูกค้า และเรื่องความเหมาะสมของราคา และเวลาดูจากความยาก ซับซ้อนของงาน ให้ลงตัว เรียบร้อย
2. วางแผนในการทำงาน โดยดูให้ตรงความต้องการของลูกค้าเป็นหลัก และสามารถทำเสร็จในเวลา 
3.การสร้างเว็ปไซต์ ทั้งในฝั่ง Front-end และ Back-end โดยทำตามแผนงาน และการตรวจสอบ อย่างเช่นการ debug หรือ การตรวจสอบเรื่องความปลอดภัยของเว็ปไซต์ ก่อนที่จะส่งงาน
4.การส่งงานในลูกค้า อาจต้องมีการแก้ไขงานหรือไม่ ขึ้นอยู่กับเงื่อนไขที่ได้ตกลงกันไว้ในตอนแรก




ถ้าให้ผมเลือก  JS Framework / Library ผมจะเลือก React ซึ่งเป็น Java script library ซึ่งReactใช้ในการสร้าง UI ได้รับความนิยมมากในการใช้สร้าง พัฒนาเว็ปไซต์ และ Reactนี้ช่วยให้การทำงานการจัดการโปรเจคของเราง่ายเป็นระเบียบมากขึ้น ทำให้สามารถทำหน้าเพจเว็ปไซต์ที่มีความซับซ้อน และสวยงามมากขึ้นได้ นอกจากนี้ Reactยังเป็นlibrary ลดการทำงานของเว็ปเซิฟเวอร์ได้มาก เป็นแบบ SPA(single page application) หรือคือการเปลี่ยนหน้าแบบไม่ต้องรีโหลด ส่วนในการตอบคำถามข้อ5.3 ผมคิดว่าเป็นคำถามที่ตอบยากเลยครับ เพราะผมเคยฝึกเขียนแค่ Reactพื้นฐานอย่างเดียว แต่จากการหาข้อมูลผมจะเลือก Veu.js  เพราะสามารถเรียนรู้ได้ไม่ยากถ้ามีพื้นฐานจาก HMTL, Css, JavaScript และเป็น SPA เหมือน React ครับ
